## Project Overview

This is a Chrome browser extension that allows users to customize web page rendering through CSS and JavaScript generated by a local large language model (LLM) running with Ollama.

**Key workflow:**
1. User describes desired page changes via chat interface in the extension popup
2. Extension captures and sanitizes the page DOM
3. DOM and user instruction are sent to Ollama (local LLM)
4. LLM analyzes the request and generates CSS code (with optional tool use)
5. Extension applies the generated CSS to the page
6. User can save CSS for automatic injection on future visits
7. User can iteratively refine changes through continued chat

## Development Commands

### Loading the Extension
Since this is a Chrome extension, there's no build process. Load it directly in Chrome:
1. Navigate to `chrome://extensions/`
2. Enable "Developer mode"
3. Click "Load unpacked"
4. Select the `extension/` directory

### Ollama Setup (Required)

**Check if Ollama is running:**
```bash
ollama list
```

**Start Ollama server:**
```bash
ollama serve
```

**Update Ollama (Ubuntu):**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**Enable CORS for Chrome extension:**
```bash
sudo systemctl edit ollama.service
```
Add:
```
[Service]
Environment="OLLAMA_HOST=0.0.0.0"
Environment="OLLAMA_ORIGINS=*"
```
Then:
```bash
sudo service ollama restart
```

**Verify CORS is enabled:**
```bash
curl -X OPTIONS http://localhost:11434 -H "Origin: http://example.com" -H "Access-Control-Request-Method: GET" -I
```
Should return 200 OK, not 403 Forbidden.

### Testing Changes
After modifying extension files, click the refresh icon on `chrome://extensions/` for the extension, then reload the target web page.

## Architecture

### Extension Components

**manifest.json** - Chrome extension configuration
- Manifest V3
- Permissions: scripting, tabs, activeTab, storage
- Host permissions: `http://localhost:11434/` (Ollama API)

**popup.html / popup.js / popup.css** - Main user interface
- Text input for user instructions
- Model selector (populated from Ollama's available models)
- Apply button triggers LLM processing
- Save CSS button persists CSS to chrome.storage.local
- Toggle to enable/disable saved CSS per site
- Link to options dashboard

**content.js** - Content script (runs on all pages)
- Automatically applies saved CSS on page load
- Listens for messages to update CSS dynamically
- Uses `__ollama_css` style element ID

**background.js** - Service worker
- Handles communication with Ollama API at `http://localhost:11434/api/chat`
- Implements MCP (Model Context Protocol) loop for tool calling
- Extracts CSS from LLM response (expects ` ```css ... ``` ` code blocks)
- Injects CSS into target page via chrome.scripting API
- Supports up to 3 tool call iterations

**tools.js** - Tool definitions and handlers
- `identify_section_by_text`: Find DOM selectors containing specific visible text
- `identify_section_by_position`: Find DOM selectors by tag and index
- `get_dom`: Return current DOM HTML
- Helper `getUniqueSelector()`: Generates unique CSS selectors using IDs, classes, and nth-of-type

**options.html / options.js** - Dashboard for managing saved CSS
- View all saved CSS rules by URL origin
- Edit URLs and CSS content
- Add new rules manually
- Delete rules

**dompurify.min.js** - Third-party library for DOM sanitization
- Cleans HTML before sending to LLM to reduce token usage

### Data Flow

1. **User input** → popup.js captures instruction and model selection
2. **DOM extraction** → popup.js uses chrome.scripting.executeScript to clone and serialize page DOM, removes any injected `__ollama_css` style
3. **Sanitization** → DOMPurify strips unnecessary tags/attributes
4. **LLM request** → popup.js sends message to background.js with instruction, DOM, tabId, model
5. **MCP loop** → background.js:
   - Sends chat request to Ollama with system prompt, user message, and tool definitions
   - If LLM calls a tool, executes handler from tools.js, appends result to chat history
   - Repeats up to 3 times or until final response
6. **CSS extraction** → background.js extracts last CSS code block from LLM output
7. **Injection** → background.js uses chrome.scripting.executeScript to inject `<style id="__ollama_css">` into page
8. **Optional save** → User clicks "Save CSS", popup.js stores CSS in chrome.storage.local keyed by origin
9. **Auto-apply** → content.js reads chrome.storage.local on page load and applies saved CSS if enabled

### Storage Schema

Chrome storage (local):
- `{origin}` → CSS string (e.g., `"https://example.com": "body { color: red; }"`)
- `{origin}_enabled` → boolean (e.g., `"https://example.com_enabled": true`)

### Key Design Patterns

**CSS Validation**: background.js validates extracted CSS is non-empty and contains at least one selector-property pair before injection.

**Unique Style Element ID**: All injected CSS uses `#__ollama_css` to allow easy removal/replacement and exclusion from DOM sent to LLM.

**Tool Calling**: Uses Ollama's function calling API. Each tool receives `dom` as first argument, then parameters object.

**Selector Generation**: tools.js generates unique selectors preferring IDs, falling back to tag + classes + nth-of-type, avoiding hallucinated selectors.
